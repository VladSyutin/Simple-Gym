Внести обязательные изменения перед началом сборки SwiftUI-компонентов: поднять iOS bootstrap, зафиксировать deployment target под Liquid Glass и заложить базовый design system foundation по первому экрану из Figma.

### Итог

Создан воспроизводимый iOS scaffold на XcodeGen, сгенерирован SimpleGym.xcodeproj, добавлены app shell, базовые design tokens, стили, пустой home screen и foundation для Empty State и Liquid Glass CTA. Deployment target зафиксирован на iOS 26.0, а решение оформлено отдельным ADR.

### Решения

- Минимальный deployment target проекта принят как iOS 26.0, чтобы опираться на нативную поддержку Liquid Glass без fallback-веток.
- Проектный scaffold создается через XcodeGen, а не вручную, чтобы структура проекта была воспроизводимой и поддерживаемой.
- Базовый design system foundation заведен сразу в коде: tokens, text styles, chrome surface и primary Liquid Glass CTA.

### Риски

- Полноценная сборка через xcodebuild сейчас блокируется локальной проблемой Xcode: не загружается системный plugin IDESimulatorFoundation.
- Код foundation провалидирован через swiftc typecheck под iPhoneSimulator26.4 SDK, но xcodebuild нужно дочинить отдельно на уровне среды.

### Follow-ups

- Починить локальную установку Xcode и повторить полноценную сборку/запуск проекта.
- После этого начать разносить первый экран на feature-specific и reusable SwiftUI-компоненты с опорой на Figma node 44:12139.
