### Запрос

Реализовать Figma-driven flow выбора групп мышц и упражнений в листах добавления тренировки/упражнения, затем довести segmented control до нативного liquid-glass поведения и перенести итог в dev.

### Итог

Добавлен reusable ExercisePickerContent для листов добавления тренировки и упражнения, включающий переходы между группами и упражнениями, множественный выбор упражнений в рамках одной группы, предзаполнение от тренировки 16 апреля 2026 и нативные swipe actions для строк упражнений. В AddWorkoutSheet segmented control был вынесен в постоянный верхний слой, чтобы сохранить системное liquid-glass скольжение при переключении вкладок. Итог сведён в dev и запушен в origin/dev.

### Решения

- Для sheet-based segmented control выбран persistent single-instance layout: segmented остаётся одним и тем же системным Picker, а меняется только контент под ним.
- Выбор упражнений внутри одной группы переведён на Set-based состояние, чтобы пользователь мог отмечать несколько упражнений и видеть счётчик 'Выбрано: N'.
- Для exercise rows в picker flow переиспользован UIKit-backed рендер swipe action images, чтобы не дублировать кастомную swipe-графику.

### Риски

- Нативное скольжение segmented подтверждено архитектурой view identity и сборкой, но финальную UX-проверку самого листа всё ещё стоит быстро прогнать в симуляторе.

### Follow-ups

- Если следующий Figma-driven flow снова потребует segmented control внутри sheet, использовать persistent top-level control как проектный паттерн.

### Запрос

Слить рабочую ветку в `dev`, запушить изменения на GitHub и обновить project memory после доработки поведения segmented control внутри flow выбора мышц и упражнений.

### Итог

В `AddWorkoutSheet` segmented control оставлен persistent single-instance view на верхнем уровне sheet, а `ExercisePickerContent` теперь только управляет его видимостью через binding. Это сохранило нативную liquid-glass анимацию системного `.segmented` при переключении между вкладками и одновременно позволило убирать control вместе с экраном выбора групп мышц, чтобы на следующем шаге оставались только упражнения.

### Изменённые файлы

- SimpleGym/Features/Home/Views/AddWorkoutSheet.swift
- SimpleGym/Features/Home/Views/ExercisePickerContent.swift
- memory/logs/2026-05-01.md
- memory/logs/index.md
- memory/index.md

### Решения

- Для нативной анимации segmented control нельзя пересоздавать его инстанс при смене состояний sheet; дочерний flow должен влиять только на visibility/layout, а не на identity control-а.
- Для экрана выбора упражнений допустимо резервировать место под верхний segmented control через layout offset, пока сам control физически остаётся в родительском chrome.

### Риски

- Сборка подтверждает корректность layout/state wiring, но плавность скрытия и возврата segmented control стоит при необходимости ещё проверить глазами в симуляторе на реальном переходе между muscle groups и exercise list.
