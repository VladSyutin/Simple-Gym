### Запрос

Слить stacked flow создания упражнения в dev и main, запушить ветки на GitHub и обновить память проекта.

### Итог

Figma-driven nested creation flow для упражнения зафиксирован в основных ветках: внутри выбора группы мышц кнопка открытия создаёт второй sheet поверх первого, для кардио показывается только поле имени, для силовых групп добавляются два временных свитча без бизнес-логики, а созданное упражнение возвращается в исходную группу и сохраняется в тренировку. После коммита изменения слиты в dev и main.

### Изменённые файлы

- SimpleGym/Features/Home/Views/CreateExerciseSheet.swift
- SimpleGym/Features/Home/Views/ExercisePickerContent.swift
- SimpleGym/Features/Home/Views/AddExerciseSheet.swift
- SimpleGym/Features/Home/Views/HomeScreen.swift
- SimpleGym.xcodeproj/project.pbxproj
- memory/knowledge/concepts/native-ios-liquid-glass.md

### Решения

- Для nested create flow в picker-based sheets выбран системный stacked sheet pattern вместо ручной замены исходного листа.
- Созданное упражнение сразу добавляется в соответствующую группу мышц, автоматически отмечается в picker и сохраняется в тренировку только через верхнеуровневый onSave flow.

### Риски

- Свитчи 'Удвоить вес' и 'Собственный вес' пока только визуальные и не участвуют в бизнес-логике; это нужно учитывать при следующем шаге по модели упражнений.
- main уже включал незапушенные изменения из dev, поэтому итоговый push синхронизирует сразу обе основные ветки.

### Follow-ups

- При следующей итерации можно привязать силовые свитчи к доменной модели упражнения и сохранению метаданных.
- Если потребуется, стоит отдельно прогнать nested sheets в симуляторе на UX-плавность ввода и закрытия.

### Запрос

Сделать Figma-driven stacked flow для создания программы: открыть второй лист поверх `Добавление тренировки`, открывать `Добавление упражнения` поверх него и удерживать выбранные упражнения внутри program sheet.

### Итог

Во вкладке `Программы` кнопка `Создать программу` теперь открывает отдельный stacked sheet `Создание программы`, а его кнопка `Добавить упражнение` поднимает уже существующий flow выбора/создания упражнений поверх этого листа. Выбранные упражнения возвращаются обратно в program sheet, остаются в его списке и используют тот же swipe interaction, что и упражнения на домашнем экране за 16 апреля. Подтверждение через `checkmark` временно сохраняет программу локально в текущей вкладке `Программы`.

### Изменённые файлы

- SimpleGym/Features/Home/Views/AddWorkoutSheet.swift
- SimpleGym/Features/Home/Views/CreateProgramSheet.swift
- SimpleGym/Features/Home/Views/HomeScreen.swift
- SimpleGym.xcodeproj/project.pbxproj

### Решения

- Общий список упражнений со swipe-actions переиспользован между home screen и program sheet вместо дублирования логики.
- Для program creation выбран отдельный nested sheet с локальным состоянием имени и упражнений, чтобы не смешивать flow создания программы с picker-контентом вкладки `Упражнения`.

### Риски

- Сохранённые программы пока живут только в состоянии текущего листа `Добавление тренировки` и не подключены к доменной модели или постоянному storage.

Вкладка `Программы` в `Добавление тренировки` теперь следует Figma-экрану `64:4894`: после сохранения программы она появляется обычной строкой в списке, а тап по строке открывает тот же `CreateProgramSheet` уже с предзаполненными названием и упражнениями для редактирования.

### Изменённые файлы

- SimpleGym/Features/Home/Views/AddWorkoutSheet.swift
- SimpleGym/Features/Home/Views/CreateProgramSheet.swift

### Решения

- Для повторного входа в программу используется тот же stacked sheet, что и для создания, но с `initialTitle` и `initialExercises`.
- В списке программ оставлен drill-in style без вторичного detail, чтобы визуально совпасть с Figma-состоянием списка программ.
