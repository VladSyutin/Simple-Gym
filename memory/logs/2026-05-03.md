### Запрос

Сделать для верхнего chrome во всех sheet-потоках soft scroll-edge эффект вместо плоского фона, протянуть fade ниже сегмента, вернуть нативную анимацию persistent segmented control и затем слить изменения в `dev`.

### Итог

Верхний chrome переведён на soft fade с overlay/content inset, сегмент снова стал persistent instance в родительском overlay, а фон ниже сегмента теперь уходит из `backgroundPrimary` в прозрачность без лишнего tint в покое.

### Решения

- Верхний chrome реализован через overlay и top content inset, чтобы список физически заходил под blur-поверхность.
- Для системной анимации segmented indicator сохранён один persistent `Picker` в родительском sheet-контейнере.
- Участок ниже сегмента выполнен как `backgroundPrimary -> transparent` fade, чтобы до скролла не появлялась полоса другого оттенка.

### Риски

- Визуальную точность fade всё ещё лучше проверить живьём в симуляторе на светлой и тёмной теме.
- Полный `xcodebuild` в ходе работы иногда подвисал из-за локальных file metadata/git процессов, хотя финальная сборка завершилась успешно.

### Follow-ups

- При необходимости отдельно подкрутить длину top fade под фактический скролл-ритм сегмента.
- Если этот паттерн закрепится, вынести параметры top chrome в отдельные design tokens или metrics.
